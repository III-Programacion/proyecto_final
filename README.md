# 📘 Proyecto Final — Aplicación Web Empresarial (CodeHub)

---

## 📖 Introducción
Como parte de mi formación en desarrollo de software, desarrollé **CodeHub**, una aplicación web empresarial orientada a la gestión interna de proyectos, clientes y empleados. El sistema incluye autenticación de usuarios, control de roles y generación de reportes en PDF, utilizando tecnologías modernas como **Spring Boot, Thymeleaf y PostgreSQL**.

---

## 🎯 Planteamiento del problema
Las empresas de software necesitan sistemas que permitan:  
- Gestionar clientes y proyectos de forma organizada.  
- Asignar empleados según perfiles.  
- Controlar accesos según roles.  
- Generar reportes confiables.  
- Reducir el uso de papel mediante digitalización.  

---

## 🎯 Objetivos
- **General:** Desarrollar una aplicación web empresarial segura con gestión de proyectos, clientes y empleados, y reportes en PDF.  
- **Específicos:**  
  - Implementar autenticación y roles.  
  - Modelar relaciones entre entidades en PostgreSQL.  
  - CRUD completo de clientes, empleados y proyectos.  
  - Generar reportes dinámicos en PDF.  
  - Versionar en GitHub y validar APIs con Postman.  

---

## 🌱 Desarrollo sostenible
El proyecto contribuye al **ODS 12: Producción y consumo responsables**, al reducir el uso de papel mediante reportes digitales y fomentar la digitalización de procesos.

---

## 🖥️ Topología y diseño
### Inventario de herramientas
| Equipo/Insumo | Descripción |
|---------------|-------------|
| Computador | Equipo principal con Windows 11, RAM suficiente y conexión estable. |
| PostgreSQL | Base de datos relacional para clientes, empleados y proyectos. |
| IntelliJ IDEA | IDE para programar en Java y gestionar el proyecto. |
| Spring Boot | Framework para lógica de negocio, controladores y seguridad. |
| Thymeleaf | Motor de plantillas para vistas dinámicas. |
| Maven | Gestor de dependencias y compilación. |
| GitHub | Repositorio remoto para control de versiones. |
| Postman | Herramienta para probar y validar APIs REST. |

### Implementación (paso a paso)
1. Configuré el entorno en IntelliJ IDEA con Maven.  
2. Creé entidades principales: Cliente, Empleado y Proyecto.  
3. Implementé relaciones JPA (OneToMany, ManyToMany, OneToOne).  
4. Configuré seguridad con Spring Security.  
5. Desarrollé controladores REST y vistas con Thymeleaf.  
6. Generé reportes dinámicos y exportación a PDF.  
7. Probé APIs con Postman y datos en PostgreSQL.  
8. Subí el proyecto a GitHub.
